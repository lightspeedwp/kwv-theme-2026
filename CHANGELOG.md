### Fixed (transparent header ŌĆö My Account icon + cart count)
- **My Account icon was invisible** in the transparent header (homepage hero). The `woocommerce/customer-account` block was placed without `iconStyle`/`iconClass`, so it fell back to WooCommerce's defaults (`iconStyle:"default"`, `iconClass:"icon"`) and rendered `<svg class="icon">`. WooCommerce only sizes the `"line"`/`"alt"` variant's `.wc-block-customer-account__account-icon` (`1.5em`) ŌĆö the default `.icon` class gets no width/height, so the SVG collapsed to **0├Ś0**. Added a `.wp-block-woocommerce-customer-account .icon { width:1.5em; height:1.5em }` rule to `assets/styles/woocommerce.css` (DB-agnostic ŌĆö fixes the icon whether the markup uses the default or `line` style), and updated the `kwv/header-transparent` and `kwv/home-hero` patterns to use `"iconStyle":"line"` to match the other headers (`header-dark`, `header-light`, `woo-header`). Confirmed live: icon now 27.8├Ś27.8 px, white, visible on the dark hero.
- **Cart item count was white-on-white.** The `woocommerce/mini-cart` count badge is coloured at runtime by core's Interactivity store, which reads the *inherited* text/background colours of the badge and **inverts** them (badge background ŌåÉ inherited text colour, number ŌåÉ inherited background) ŌĆö overriding whatever product-count colour is set in the editor. The transparent header forces white (`base`) text for legibility over the hero, so the store rendered a white pill with a white number. Added a `.wc-block-mini-cart__badge` rule with `!important` (author-important beats the store's inline non-important styles) forcing a fixed legible pill. Confirmed live: visible count.
- **Note:** the homepage renders from a **DB-customised `front-page` template** (Site Editor override, `wp_template` post 254) with the hero + transparent-header markup baked in ŌĆö not from `templates/front-page.html` / the patterns. The CSS fixes apply regardless; the pattern-file edits keep the theme source correct for any future sync/reuse but do not themselves change the live homepage.
