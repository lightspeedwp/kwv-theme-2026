# KWV E2E — Playwright tests

End-to-end tests for the KWV 2026 WooCommerce site, generated by the LightSpeed
**Playwright Testing Agent** from an approved test pack. Self-contained: install
and run from this directory.

- **Test pack (source of truth):** `reports/test-packs/checkout-2026-07-24.md` (workspace root)
- **Linear:** [LS-1175 · 15. Testing & Launch](https://linear.app/lightspeedwp/issue/LS-1175/15-testing-and-launch)
- **Flow covered this round:** cart → mini-cart → cart page → checkout → payment

## Setup

```bash
cd wp-content/themes/kwv-theme-2026/tests/e2e
npm install
npx playwright install --with-deps chromium
cp .env.example .env   # then fill values (never commit .env)
```

## Run

```bash
npm test                 # all non-@stateful cases (TC-001–009)
npm run test:cart        # cart specs
npm run test:checkout    # checkout specs
npm run report           # open the HTML report

# @stateful (order-placing) cases run only when the env is sandbox-ready:
#   KWV_SANDBOX=1 and KWV_TEST_CARD_NUMBER set (see .env.example)
```

## Environment & Test-Data Contract

All config comes from `.env` (see `.env.example`). Never hard-code secrets or real
customer data. `@stateful` cases (order-placing) **skip** unless `KWV_SANDBOX=1`
and a test card are present — they must never run against production.

## Conventions

- **Accessible locators** (`getByRole`/`getByLabel`) first.
- **WooCommerce Blocks** checkout; the **mini-cart is a body-level portal**
  (`.wc-block-mini-cart__drawer`).
- **Store API** recalculates async — specs wait on the mutation/updated element,
  not fixed timeouts. Staging is slow; timeouts are generous.
- **`@stateful`** tags every order-placing case.
